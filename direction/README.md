## 4.大数据/数据挖掘/内容检索

大数据研发，简单来说，就是围绕海量数据，搭建数据平台、进行数据存储与处理、提供分析与服务。

目标：让数据 可采集、可存储、可计算、可分析、可应用。

大数据研发 =

👉 用 **存储** 保存数据

👉 用 **集成** 把数据拉进来、推出去

👉 用 **计算** 加工数据

👉 用 **调度/安全/运维** 保证系统可控

👉 用 **平台和中间件** 提升效率

👉 最终把数据转化为 报表、看板、标签、主题分析、推荐、搜索、智能应用

常见数据组件如下：
+ **数据存储（解决“数据放哪里”的问题）**
  + HDFS（分布式文件系统）
  + Hive（离线数仓）、MaxCompute（云离线数仓）
  + HBase（时序存储NoSQL）
  + ClickHouse（实时分析型数据库）、AnalyticDB（阿里云实时分析型数据库、实时数仓）
  + S3（对象存储）、OSS（阿里云对象存储）、OBS（华为云对象存储）
  + Oracle（商业数据库）、Mysql（开源关系型数据库）、GaussDB（华为国产化替代分布式数据库）
+ **数据集成（解决“数据怎么来”的问题）**
  + Kafka（消息队列）、DataHub（实时数据总线）
  + DataX（离线数据同步）、Sqoop（离线数据同步）、DataWorks-DI（阿里云离线数据同步）、DRS（华为云离线数据同步）
  + Flink CDC（实时流数据同步）
  + Oracle GoldenGate（Oracle实时数据复制与同步）
+ **数据计算（解决“数据怎么加工”的问题）**
  + Spark（批处理计算）
  + Flink（流处理计算）
+ **监控、运维、安全和调度（保证系统 稳定、安全、可控）**
  + Zookeeper（协调服务）
  + Kerberos（安全认证）
  + SchedulerX（任务调度）
  + Azkaban（工作流调度）
+ **云数据开发平台（让研发和数据分析更高效）**
  + DataWorks（阿里云）
  + DataArts Studio（华为云）
+ **中间件（为大数据系统提供支撑）**
  + RocketMQ（阿里开源金融级消息队列）、RabbitMQ（轻量级消息队列）

## 10.交叉、综合、新兴

+ 区块链.
    区块链技术的社区建设其实会比我们想象的丰富很多，各种 `DeFi`、`GameFi` 等非常有创造力的 `DApp` 在区块链上诞生。  
    如果有感兴趣入门的同学，我非常推荐这个课程：[僵尸游戏](https://cryptozombies.io/zh/course)。  
    里面会把最基本的区块链常用于编写合约的语言 `Solidity` 做了非常全面的介绍，做一个 `DApp` 开发的流程，也会在一些安全性上，  
    如何减少成本（减少`gas`费用）上做出一些实践后的建议，个人觉得非常值得入门。（但最好要有其他编程语言基础再看，最好是 `C++`。非常好上手！(dumengrong)
